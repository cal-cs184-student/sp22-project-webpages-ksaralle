Final Project Proposal: Rendering mods for Games

Bohan Yu, Yao Fu Zuo, Sara Wang, Hao Jie Zhou



Summary

In our final project, we will be implementing path-tracing and different materials (e.g. mirroring, refraction, etc) in Minecraft to make the game more beautiful without sacrificing too much performance. We would play the game with the shader packages we’ve engineered, and we would compare the performance and the quality of the game scenes against those of the default packages.

Problem Description

Currently, the shaders in Minecraft are either too ugly or too computationally demanding due to lack of optimization, though some of them are due to very detailed textures. For example, the default texture—without any path-tracing—renders unrealistic torch, water, clouds, etc. The block-by-block lighting also adds lots of sharpness instead of smoothness to the game. In our final project, we will construct a considerably refined implementation of shaders that not only provides more delicate rendering, but also good runtime performance.

There are several challenging aspects of the project. First of all, it takes effort to depict a sense of realism in an abstract game environment that is constructed by blocks. However, we can rely on the physics of light reflections, refractions, and techniques of ray tracing to simulate real-world sophistication.

Second, since we are doing more complex graphics computations, keeping the time performance to an acceptable extent will also be a challenge. We can reduce the runtime burden by also making an optimization of the implementation with respect to time complexity.

Goals and Deliverables

We will create images with different scenes showing how different elements of the world have been modified to increase graphical fidelity. For example, we would like to show images to compare the vanilla version of the game’s water to our version of the game’s water.

To measure performance, we can render a static environment within the game and record the frames per second when running different/no shaders.

What questions do you plan to answer with your analysis?

Why might people consider altering the graphics of a game?

How large of an improvement can optimizations(like using Stacks instead of recursion) make to rendering Minecraft scenes?

What visual improvements can we make without impacting game performance given our devices/technology today?

What materials could be visually the best choice for different minecraft blocks?

What we plan to deliver

Good choices of parameters to simulate realistic materials. Textures and materials show up properly: For example, surfaces should be sampled from real-world materials with realistic BRDFs.

Materials rendered correctly. For example, with the glass mirroring the light and the water refracting the light, the output value should be correct. (E.g. Reflecting the light should not make it brighter.)

Several objects that are the highlight of our visualization efforts: For example, glazing torch / candles, sophisticated shadows.

The game runs smoothly with path-tracing (>30 FPS AVG on a modern GPU (NVIDIA 3070 Ti))

What we hope to accomplish

When rendering scenes with similar quality, the performance of our shaders packages is more efficient than other online packages.

Since our shaders packages would be written mainly in OpenGL, we hope to apply this to other games that use the OpenGL library while using other modification injection tools such as ReShade.

Why we think we can accomplish those goals

The visual effects are supported by physics and the existing resources on the internet allow us to understand the inner workings of the game to the extent that we need in order to create these modifications. The game also uses fairly standard graphics libraries and common programming languages that we are either familiar with or are interested in learning.
